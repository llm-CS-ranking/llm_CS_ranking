# LLMs as Cross-Sectional Stock Rankers

> A reproducible benchmark for testing whether frontier LLMs behave like factor-like priors in equity ranking.

[![Python](https://img.shields.io/badge/Python-3.10%2B-blue)](#quickstart)
[![LaTeX](https://img.shields.io/badge/Paper-ACM%20LaTeX-purple)](#paper)
[![Reproducibility](https://img.shields.io/badge/Reproducibility-tables%20%2B%20figures-brightgreen)](#reproduce)
[![Finance](https://img.shields.io/badge/Finance-LLM%20ranking-orange)](#what-this-repo-does)

This repository studies a simple but surprisingly powerful question:

**If every model sees the same point-in-time financial information, do different LLMs induce different stock rankings, factor exposures, and long-short returns?**

Instead of asking models to forecast noisy return magnitudes, the benchmark asks them to produce a monthly **listwise ranking** of a fixed NASDAQ universe. Those rankings are converted into quintile portfolios, transaction-cost-aware long-short spreads, rank ICs, Fama-French six-factor spanning tests, input ablations, and within-universe style fingerprints.

The result is a compact benchmark for treating LLMs not as chatbots, but as **implicit cross-sectional priors**.

## Why This Is Interesting

- **Listwise, not vibes.** Models output full stock orderings, enabling portfolio diagnostics rather than anecdotal picks.
- **11 frontier LLM rankers.** The benchmark compares OpenAI, Google Gemini, and Anthropic Claude families under a common protocol.
- **Baselines included.** Non-LLM factor and ML rankers are evaluated in the same common-window setup.
- **Factor-model lens.** Outputs are tested with FF6 spanning regressions and within-universe characteristic mimicking portfolios.
- **Input ablations.** The repo separates the roles of fundamentals, market summaries, and their combination.
- **Paper-ready pipeline.** Tables and figures are generated by focused modules, so future agents can rerun only the artifact they changed.

## Headline Findings

In this benchmark window, the strongest expected-return rankers produce large positive Q1 and Q1-Q5 spreads, while weaker models can flip the sign of the signal. The stronger rankers share a growth-momentum tilt, but their FF6 loadings and within-universe style correlations differ by vendor and generation.

The main empirical lesson is not “deploy this as a trading strategy.” It is that controlled LLM rankers can be evaluated as **model-induced factor priors** with standard asset-pricing tools.

## Repository Layout

```text
.
├── llm_ranking/              # Refactored package: paths, config, IO, builders, renderers, CLIs
│   ├── cli/                  # Minimal commands for building results and rendering artifacts
│   ├── evaluation/           # Cached-panel -> CSV metric builders
│   ├── io/                   # Market/panel loading helpers
│   └── rendering/            # LaTeX table and figure renderers
├── scripts/                  # Thin wrappers and experiment entrypoints
├── paper/                    # Main-paper LaTeX source, figures, tables, and compiled PDF
├── paper_appendix/           # Extended appendix LaTeX source, figures, tables, and compiled PDF
├── result/
│   ├── tables/               # Publication CSV/TEX artifacts
│   └── common_window/        # Common-calendar robustness artifacts
├── conf/                     # Public config; API keys are read from environment variables
├── utils.py                  # Portfolio, prompt, payload, and ranking utilities
└── factor_utils.py           # FF factor and characteristic-analysis utilities
```

## Quickstart

Create an environment:

```bash
python -m venv .venv
source .venv/bin/activate
pip install -r requirements.txt
```

Render all LaTeX tables from included CSV artifacts:

```bash
python -m llm_ranking.cli.render_tables --only all
```

Render included CSV-based figures:

```bash
python -m llm_ranking.cli.render_figures --only fig9
python -m llm_ranking.cli.render_figures --only fig10
```

Compile the papers:

```bash
(cd paper && latexmk -pdf -interaction=nonstopmode -halt-on-error paper.tex)
(cd paper_appendix && latexmk -pdf -interaction=nonstopmode -halt-on-error paper.tex)
```

## Reproduce

The public release includes the paper-facing CSV/TEX artifacts needed to regenerate tables and selected figures:

```bash
python -m llm_ranking.cli.render_tables --only table10
python -m llm_ranking.cli.render_tables --only common_window
python -m llm_ranking.cli.render_tables --only appendix_metrics
python -m llm_ranking.cli.render_tables --only factor_tables
```

Some builders require raw market/fundamental data and cached rank-panel pickles, which are intentionally omitted from this anonymized public package:

```bash
python -m llm_ranking.cli.build_results --only common_window
python -m llm_ranking.cli.build_results --only input_ablation
python -m llm_ranking.cli.render_figures --only performance
```

If you have the same data layout locally, place it under:

```text
data/ndx_rolling_20260107/
  ndx_market_data.csv
  ndx_fundamental_data.csv
  ndx_tradingiteminfo.csv
  ndx_data_member.csv
```

LLM rank-panel outputs are expected under `result/` and `result/ablations/`.

## Running New LLM Calls

API-key files are not committed. The public config reads credentials from environment variables:

```bash
export OPENAI_API_KEY="..."
export GEMINI_API_KEY="..."
export ANTHROPIC_API_KEY="..."
```

Always sanity-check an experiment with dry-run mode first:

```bash
python scripts/run_llm_ranking_experiment.py \
  --model gpt-4.1-nano \
  --modes return \
  --input-set both \
  --max-months 1 \
  --dry-run
```

Run input-ablation batches without API calls:

```bash
python scripts/run_input_ablation_experiments.py \
  --models gpt-4.1-nano \
  --input-sets market_only \
  --modes return \
  --max-months 1 \
  --dry-run
```

## Design Principles

This repo is organized around one practical rule:

**Expensive experiments, medium-cost metric builders, and cheap renderers should be separate.**

That lets you modify one table caption, one bolding rule, or one figure style without accidentally rerunning LLM inference. The most common paper-polish commands are now narrow and explicit:

```bash
python -m llm_ranking.cli.render_tables --only table10
python -m llm_ranking.cli.render_tables --only common_window
python -m llm_ranking.cli.render_figures --only fig10
```

## What Is Not Included

- Raw licensed market/fundamental data.
- Private API keys.
- Cached LLM rank-panel pickle files.
- Per-call API metadata and retry logs.
- Local LaTeX build products such as `.aux`, `.log`, `.fls`, and `.fdb_latexmk`.

## Disclaimer

This repository is for research and reproducibility. It is not investment advice, not a live trading system, and not a recommendation to trade any security. Reported portfolio statistics are benchmark diagnostics over a narrow universe and time window.
